# OSINT Expert Roadmap & Methodology

A complete, realistic and ethical framework to learn, practice and professionalize
Open Source Intelligence (OSINT), from beginner to expert level.

---

## About this project

This repository provides a structured and progressive OSINT framework.
It goes beyond simple information gathering and focuses on:

- analytical rigor
- verification and validation
- ethical and legal boundaries
- professional reporting

The goal is to help learners transition from OSINT hobbyist to credible OSINT analyst.

---

## Intended audience

This repository is designed for:
- OSINT learners (beginner to advanced)
- Journalists and researchers
- Cybersecurity and threat intelligence professionals
- Anyone interested in responsible open-source investigations

This repository is not intended for:
- Doxxing or harassment
- Illegal data collection
- Privacy violations
- Unethical OSINT practices

---

## Core principles

This project is built on the following principles:
- Legality before capability
- Ethics before curiosity
- Verification before conclusion
- Methodology before tools
- Documentation before intuition
- Acceptance of uncertainty

OSINT is not about finding information.  
It is about producing reliable and defensible intelligence.

---

## Roadmap overview

The learning path is divided into six progressive phases, detailed in the main roadmap document.

1. Foundations OSINT  
2. SOCMINT and digital identities  
3. GEOINT and verification  
4. Automation and technical OSINT  
5. Strategic analysis and professional OSINT  
6. Specialization and validation  

Each phase includes objectives, methodological expectations and concrete deliverables.

## How to use this repository

1. Start with the main roadmap document
2. Apply the methodology during exercises or investigations
3. Use templates and checklists systematically
4. Separate facts, hypotheses and judgments
5. Document sources and reasoning
6. Seek external validation of your work

This repository is meant to be used as a working framework, not read once.

---

## Skill validation

OSINT expertise requires external validation.

Recommended activities include:
- OSINT challenges and CTFs
- Reproducing public investigations
- Peer review and collaboration
- Publishing documented work

Without validation, OSINT skills remain theoretical.
